What I Do

I am passionate about application development, hardware trends, and using technology to better customer experience.

Here is a sample of my projects

MoneyPLS Event Planner

Created a event money manager web app using the Venmo API. Hosts create an event and send out a payment link to guests. Invitees login with their Venmo credentials to pay. Hosts can keep track of who's paid and total money raised. Won Best Use of Azure Cloud Services from Microsoft at annual TartanHacks hackathon at CMU.

Dynamic Memory Allocator

Created a version of the standard C malloc package. Implemented doubly linked segregated lists to efficiently store and free data. Includes the functions Malloc, Free, Calloc, and Realloc. Achieved 51% util and 41% throughput.

Project Pond

Created a location based file sharing service for Penn Apps Fall Hackathon. Users can upload a file and specify a radius other users can see the file, the lifetime of the file after which it will disappear, and a password for the file. The idea was to have anonymous file sharing with minimal authentication needed. Used MongoDB, Django, and GridFS.

C0 Virtual Machine

The language c0 is a simplified introductory subset of C used in CMU classes. I implemented a virtual machine in C to parse and execute arbitary c0 programs. It used a stack and heap approach to organize instructions.

Mayur SASA Website

As Webmaster, I developed and currently maintain the website for CMU's South Asian Student Association. Check it out at www.mayursasa.com

Motion Replication

Developed a motion replication application during Build 18, a software/hardware hackathon at CMU. I used a Leap Motion to track hand movements and emulated the movements to move a piece of acrylic. I sent signals to the Arduino to move stepper motors attached with fishing line to the board.

8-bit Mastermind

I implemented the classic game MasterMind in x86 Assembly and SystemVerilog HDL on an FPGA board. It features a fully functional 8-bit GUI as well as a computer player. Interesting fact: This was done without the use of IF statement (hardware programming is tough).

MongoDB Inc

Software Engineering Intern

I worked and collaborated with full-time engineers as part of the Cloud Manager Core team

Cloud Manager is MongoDB's enterprise software solution to monitor, automate, backup, and configure MongoDB deployments, from standalones to multi-sharded Clusters. It's used by thousands of customers in a variety of ways.

I helped implement new features mainly focusing on improving the Cloud Manager Alert system. Customers can set alerts to notify them if some problem occurs with their deployment i.e. a host is down or opcounters for a server or above/below a certain level. I integrated Slack and Flowdock (increasingly popular team chat platforms) so that alerts can be sent directly to team chat rooms.

Another key feature I implemented was maintenance windows for sys admins. This allows admins to schedule maintenance on their servers and not be inundated with false alerts. This was a completely new feature and as a result I had considerable power over architecture and design decisions.

Throughout the summer I primarily worked with Java in the back-end and Backbone.js in the front end. The job involved a lot of refactoring of old legacy code. It was a very rewarding and exciting experience.

KeyMe Inc

Software Engineering Intern

I worked with a team of engineers to setup, configure, and automate the next generation of the key copying kiosk

To automate kiosk dependancy setup, I used Salt, a configuration management tool, to remotely deploy all relevant dependancies. I first used Vagrant and VirtualBox to simulate the system, which allowed me to test my Salt tools. This resulted in a dramatic decrease in setup time and cost

Additionally, I created a series of Python application to test various components and systems inside the kiosk including various motors, cameras, and I/O boards.

Teaching Assistant

Excerpt from the website: "The purpose of this course is to introduce the theory and practice of functional programming (FP). The characteristic feature of FP is the emphasis on computation as evaluation. The traditional distinction between program and data characteristic of imperative programming (IP) is replaced by an emphasis on classifying expressions by types that specify their applicative behavior. Types include familiar (fixed and arbitrary precision) numeric types, tuples and records (structs), classified values (objects), inductive types such as trees, functions with specified inputs and outputs, and commands such as input and output. Well-typed expressions are evaluated to produce values, in a manner that is guaranteed to be type-safe. Because functional programs do not cause side-effects we can take advantage of simple mathematical principles in reasoning about applicative behavior and analyzing the runtime properties of programs."
